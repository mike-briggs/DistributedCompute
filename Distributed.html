Benefits of the Distributed Computer

By creating a global supercluster from both dedicated and underutilized compute resources, DCP lets you build for a scale previously limited to large corporations. Thousands of CPU and GPUs provide the virtual infrastructure to easily accelerate your compute-intensive work both securely and seamlessly.

Massively Parallel Execution

Access more CPU and GPU cores for accelerating your compute-intensive code. DCP's serverless architecture lets you scale up and speed up as easy as changing an integer value.

Write For Anywhere

Traditional microservices and serverless compute locks you into a specific architecture. The Distributed Computer lets you skip across different infrastructure with the same code.

Hybrid Environment

With identical implementation across different architectures, DCP lets you seamlessly burst to a WAN compute cluster from a local one. Different clusters can also seamlessly combine and seperate.

An Open App Economy

Developers can make the most of applications with embedded distributed compute. The DCP application market provides pre-made software modules, as well as rich SDKs for all purposes.

Cross-Platform Data Security

DCP's data sharding algorithms mean that high-risk data is sent to verified data centers, while kept in secure isolation from other workloads. Unlike the central cloud, data is not vulnerable at rest.

Optimize Utilization

Data within the same set might vary widely in size and structure. The Distributed Computer matches tasks with the right hardware to ensure memory and compute requirements are met without waste.

The Distributed Computer is a pool of serverless compute resources across various nodes and machines. When a developer initiates a workload through this network, it is characterized and then matched with the appropriate type of hardware.

A single workload can be any size, from a single virtual thread running inside an isolated sandbox to multiple datacenters in different cities. All compute is executed inside the same secure JavaScript V8 engine as local DCP instances.

Economics of the Compute Economy

As data and compute requirements continue to rise exponentially, there is tremendous demand for scarce compute resources. The Distributed Computer refactors the economics of compute to favor the developer, not the corporation.

Increased Compute Supply

The Distributed Compute securely connects every digital device from gaming GPUs to x86 servers. Public clouds combined with previously private networks are all accessible, representing more compute than the largest providers individually.

Perfectly Competive Environment

Compute is fungible with write once, run anywhere functionality. There are no special characteristics to make any one provider superior from the developers view, making the compute market productively and allocatively efficient.

Transparent Price Discovery

DCP's scheduling algorithms quantify the true economic costs of compute, and put participants on an equal playing field. This allows the true intersection of supply and demand to be known, giving developers more dedicated

Contribute Compute Power

The Distributed Computer is a two-sided marketplace. Join and earn compute as easily as opening a web page - DCP workers operate through a browser, or they can act as standalone Node.js daemons for Linux, Windows, and OS X.

By powering someone else's applications during your hardware's idle period, you could be accelerating your own in periods of compute-intensive load. You may operate on an entirely volunteer basis, or set an amount to cover the cost of your network and compute resources.

Forced Capacity Limits

DCP executes on your local device in a completely non-interference way. You can set it to use only a portion of idle capacity, work only during certain hours, or both.

Fine-Grained Control

You can choose which applications to contribute compute power to. Favor computational research from a certain institution, only process Canadian datasets, and more.

Local Payouts

DCP accounts for compute with real currency. You can use the value that is accumulated to boost your own workloads, or request a payout in through your chosen method.
