Technology

The DCP stack is built from the ground up to improve compute efficiency and scalability as simply as possible. It reflects classic elements of distributed computing with a new vision to make compute fungible across any hardware.

Learn how DCP's architecture reimagines distributed computing to connect millions of heterogeneous devices together.

Simplified Programming

DCP uses the JavaScript V8 engine to execute all computations. Unlike other languages, this enables JavaScript code to run on any device.

Fungible Infrastructure

DCP leverages WebAssembly to provide multiple performance advantages and in the future to provide compatibility with other languages like Python, C, and Rust.

Virtual Clusters

DCP integrates WebGL 2.0 for rendering and general computing tasks. It enables GPU processing and, in the near future, General-Purpose GPU compute (GPGPU).

The Tech Stack

DCP is an intercoupled set of software modules that forms a lightweight overlay network. The individual modules are primarily built on open-source frameworks, with a few novel tweaks to improve the utilization of computing resources.

Why Javascript

JavaScript is the universal language in which all websites and web applications are written. Although it is not often thought of as a tool for compute-heavy jobs, there are several reasons that make it the obvious choice for a distributed compute network:

  • It is nearing the speed and effectiveness of low-level languages like C++, while being more performant than a high level language like Python.
  • JavaScript and it's engine performance is constantly being improved by Google, Mozilla, Apple, and Microsoft!
  • WebAssembly allows JavaScript to benefit from the convenience of being a high-level language, but have the performance of low-level ones.
  • JavaScript has the largest and youngest development community and is the most popular language - NPM is the single largest software repository!
